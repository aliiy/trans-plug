## 📋 沉浸式翻译插件 - 最终需求确认书

### 一、技术栈与分工
| 角色 | 工具/服务 | 职责 |
|------|-----------|------|
| **代码生成** | OpenCode + GLM-5.1 | 编写全部插件源码、项目结构、调试 |
| **翻译引擎** | DeepSeek-V4-Flash API | 接收文本数组，返回译文（JSON格式） |
| **运行环境** | Chrome / Edge 浏览器（Manifest V3） | 拦截网页、渲染译文、存储配置 |

---

### 二、核心功能规则

**1. 语言方向**
- **源语言**：自动识别（英语 / 日语）
- **目标语言**：简体中文（固定）
- **识别机制**：不前端判断，全靠 DeepSeek System Prompt 自动处理

**2. 翻译范围**
- ✅ **翻译**：块级元素（`<p>`、`<div>`、`<li>`、`<article>`、`<section>`、`<h1>` ~ `<h6>`）
- ❌ **禁止触碰**：`<video>`、`<audio>`、`<canvas>`、`<svg>`、`<code>`、`<pre>`，以及 `role="button"` 或 `contenteditable="true"` 的容器

**3. 渲染样式（对标 Immersive Translate）**
- 原文块**完全不动**，在其**正下方**插入独立译文块
- 译文块样式：
  ```css
  背景: #f7f8fa (极浅灰)
  左边框: 4px solid #1e6ef0 (蓝色竖条)
  内边距: 8px 14px
  字号: 0.9em (略小)
  圆角: 0 4px 4px 0
  淡入动画: opacity 0→1 (0.3s过渡)
  ```

---

### 三、核心交互机制（最关键——滚动触发）

| 行为 | 具体逻辑 |
|------|----------|
| **初始加载** | 抓取全页面所有块级元素，**不翻译**，仅存入队列并绑定 `IntersectionObserver` |
| **滚动触发** | 当块进入视口（提前 **300px** 预加载），加入翻译队列 |
| **批量发送** | 用 `Set` 去重 + **200ms 防抖**，每批打包 **10~15 个块** 发给 DeepSeek |
| **SPA 动态内容** | `MutationObserver` 监听新增块，对新块同样绑定 Observer |
| **开关控制** | Popup 关闭时，`observer.disconnect()` + 一键移除所有 `.imm-trans-block` |

---

### 四、性能优化策略

| 优化项 | 方案 |
|--------|------|
| **API 调用** | 批量请求（减少并发），流式输出（`stream: true`）可选，第一版建议关掉先跑通 |
| **缓存** | 原文取 MD5 哈希存入 `chrome.storage.local`，命中则直接渲染，不走 API |
| **防卡顿** | 用 `requestIdleCallback` 调度非紧急任务 |
| **安全性** | 禁止 `innerHTML`，一律 `createElement` + `textContent` |

>
> **核心机制**：
> 1. 抓取块级元素（p, div, li, article, section, h1~h6），过滤 video/audio/canvas/svg/code/pre 及 role="button"/contenteditable 容器。

> 4. 译文以独立块（灰底蓝边，淡入动画）插入原文正下方（`afterend`），原文保持不变。
> 5. 缓存命中（chrome.storage.local）则直接渲染，不调用 API。
> 6. MutationObserver 监听 SPA 新增块，同样绑定 Observer。
> 7. Popup 面板含 API Key 输入、开关、清除缓存。
>
> **严禁**：使用 innerHTML，翻译标题（已包含在 h1~h6 中），触碰视频相关元素。
